Umfrage zur Weiterentwicklung von CrypTool

CrypTool wurde bisher unter Visual C++ v6 mit den Microsoft Foundation Classes (MFC) entwickelt. Dies war beim Start des Projektes 1996 eine gute Wahl. Für eine zukunftsgerechte und moderne Weiterentwicklung wollen wir auf eine neue Entwicklungsplattform umsteigen.

Derzeit finden Überlegungen statt, CrypTool auf Java oder C# zu portieren. Wir möchten CrypTool-Interessierte bitten, uns Ihre Meinung zu dieser Thematik mitzuteilen.

Visual C++ / MFC (wie bisher)
Java / Swing
Java / SWT (mit Eclipse)
C# / .NET / Windows Forms
C++ v7 / .NET / Windows Forms

  • Ihre Gründe
