Padding

Padding means that the data to be processed - for instance a document that is to be encrypted - is padded in a certain way. Padding mechanisms are an important part of modern cryptography.

Hash functions process data in blocks of fixed size. Therefore, the data to be hashed is padded according to certain rules. Likewise, this is the case with some symmetric encryption algorithms. They require the length of the data to be encrypted to be multiples of a certain block size. Two examples for these algorithms are DES (CBC) and Triple-DES (CBC), both using the Cipher Block Chaining mode.

Depending on the encryption mode used, it is possible that a document of zero byte length is longer after encryption.

CrypTool uses the following padding methods for block ciphers. As an example we assume that the message 61 62 63 shall be padded to 8 bytes.

PKCS #7
The number of required padding bytes (> 0) is encoded with binary encoding in one byte. This byte is used to pad the message.
Example: 61 62 63 05 05 05 05 05
Zero
Appends (if required) zero bytes until the message is padded to a multiple of the block length. This padding method can only be inverted unambiguously if the message does not end in zero bytes.
Example: 61 62 63 00 00 00 00
ANSI X.923
Appends zero bytes, followed by a byte containing the number of message bytes in the final block in binary encoding. In a different variant the last byte contains the number of padding bytes. CrypTool uses the former alternative.
Example: 61 62 63 00 00 00 00 03
01 00 ...
Appends the byte 01 and fills the rest of the block with zero bytes.
Example: 61 62 63 01 00 00 00 00
