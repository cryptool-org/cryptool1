CrypTool PKCS#12 import/export documentation

CrypTool supports exporting and importing of digital certificates with the corresponding private key in a PKCS#12 structure. PKCS#12 is a widely adopted industry standard that allows safe delivery of sensitive data. The private key contained in a PKCS#12 structure is protected by symmetrical encryption; the key for this encryption is computed from a transport password that protects the PKCS#12 structure.

CrypTool is able to export its own private keys and corresponding X.509 certificates in a PKCS#12 structure, allowing these certificates to be imported in other CrypTool installations. It is possible to import and use these PKCS#12 files in other applications as well (such as Netscape or MS Windows).

Users should be aware that the certificates created by CrypTool are for educational use only. It is not recommended to use these certificates in real-world applications because of security considerations (the CrypTool CA key is the same for each CrypTool installation).

Limitations:

Although the certificates created and used in CrypTool are standard X.509 certificates, it is currently not possible to import foreign certificates into CrypTool. CrypTool stores internal data in the certificate subject name that is not present in foreign certificates. This data is necessary to fill internal CrypTool data structures when transferring certificates to another CrypTool installation.
