Solitaire analysis (dialog usage) (Menu Analysis/ Symmetric Encryption (classic) / Ciphertext only)

This dialog provides a cryptanalysis (Solitaire analysis) of a ciphertext, which is encrypted with the Solitaire algorithm The ciphertext has to be loaded as a text document. The text is bounded by 65.535 letters. The rest will be ignored. Then the initial deck is generated by using the found final deck. With this initial deck the keystream will be generated and the ciphertext can be decrypted.
Please follow the numbers of the frames (groupings) in this dialog (grouping 1 to 4).

Determining the initial deck

Grouping 1: Parameters of the found deck
Please choose the way of determining the found final deck. For that click one of the buttons Load final deck or Own choice.

  • Load final deck

    • You load a final deck, which has been saved at the encryption.
  • Own choice

    • You create your own final deck in the grouping "Own choice". Also the number of cards can be variied.

Grouping: Own choice
This grouping becomes active, when you chose "Own choice" in grouping 1. First you have to choose the number of cards (3 to 54). 54 is setted as the default value. Then you construct your prefered final deck through clicking the buttons 1 to 54 in your wished order. If you make a mistake, please click the Reset button and restart your enter. While you are clicking the buttons, you can see the current deck in the output field "Found deck" in grouping 3.. When you reach the chosen number of cards, this grouping becomes inactive and grouping 2 becomes active.

Grouping 2: Positioning of the joker
Because of the irreversibility of the Solitaire algorithm in some special cases it has to be decisioned, whether to place joker A or Joker B at the top or at the bottom of the card deck. This grouping allows to choose between Always to front, Always to back and Start manual.

  • Always to front

    • It is tried to generate the initial deck. If a special case appears, the joker is always placed to the front of the card deck.
  • Always to back

    • It is tried to generate the initial deck. If a special case appears, the joker is always placed to the back of the card deck.
  • Start manual

    • It is tried to generate the initial deck. If a special case appears, the user has to decide where to place the joker (in the front of the card deck or in the back of the card deck). With the button Cancel in the Message Box you can leave the procedure.
The manual cryptanalysis as well as the automatical cryptanalysis can take wrong decisions, with the result of a wrong plaintext.

Grouping 3: Generating the initial deck
This grouping provides to applicate the differnet states of the card decks.

  • Found deck

    • Represents the found card deck, which is either loaded or created by the user.
  • Temporary deck

    • Represents the different modifications of the card deck during the cryptanalysis.
  • Initial deck

    • Is the cryptanalysis successfull, the same initial deck as the one, which has been used for generating the keystream at the encryption, is shown here.

Grouping 4: Generating the keystream
By using the created initial deck the Solitaire algorithm can be started by clicking the button Automatic generating of the keystream. So the keystream is generated and it is possible to decrypt the ciphertext.

Decrypt
If the keystream is generated completely, the ciphertext can be decrypted by clicking the button Decrypt (but only if the created initial deck is the same one, as the initial deck, which was used for the encryption).

Cancel
With this button you can always leave the dialog.
