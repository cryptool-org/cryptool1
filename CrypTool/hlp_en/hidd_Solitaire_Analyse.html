Dialog Solitaire analysis

This dialog provides a cryptanalysis (Solitaire analysis) of a ciphertext, which is encrypted with the Solitaire algorithm. The ciphertext has to be loaded as a text document. The text is bounded by 65,535 letters. The rest will be ignored. Then the initial deck is generated by using the found final deck. With this initial deck the keystream will be generated and the ciphertext can be decrypted.

Please follow the numbers of the frames (groupings) in this dialog (grouping 1 to 4).

Determining the initial deck

Grouping 1: Parameters of the found deck

Please choose the way of determining the found final deck. For that click one of the buttons Load final deck or Own choice.

  • Button Load final deck: You can load a final deck, which has been saved at the encryption.
  • Button Own choice: You can create a new final deck in the grouping "Own choice". Also the number of cards can be varied.

The grouping Own choice becomes active, when you chose "Own choice" in grouping 1. First you have to choose the number of cards (3 to 54). 54 is set as the default value. Then you construct your prefered final deck through clicking the buttons 1 to 54 in your wished order. If you make a mistake, please click the Reset button and enter your input again. While you are clicking the buttons, you can see the current deck in the output field "Found deck" in grouping 3. When you reach the chosen number of cards, i.e you clicked all buttons, you get an according message.


Grouping 2: Positioning the joker

Because of the irreversibility of the Solitaire algorithm in some special cases it has to be decided, whether to place joker A or joker B at the top or at the bottom of the card deck, when trying to find the initial deck. This grouping allows to choose between Always to front, Always to back, and Start manual.

  • Always to front: If a special case appears, the joker is always placed to the front of the card deck.
  • Always to back: If a special case appears, the joker is always placed to the back of the card deck.
  • Start manual: If a special case appears, the user has to decide where to place the joker. With the button Cancel in the message box you can leave the procedure.

The manual cryptanalysis as well as the automatical (using always the same decision) cryptanalysis can take wrong decisions, with the result of a wrong plaintext.


Grouping 3: Generating the initial deck

This grouping provides three different card decks:

  • Found deck: Represents the found card deck, which is either loaded or created by the user. It should be a valid final deck created when performing an earlier encryption with Solitaire.
  • Temporary deck: Represents the different modifications of the card deck during the cryptanalysis.
  • Initial deck: If the cryptanalysis is successful, the same initial deck as the one, which has been used for generating the keystream at the encryption, is shown here.

Grouping 4: Generating the keystream

By using the created initial deck the Solitaire algorithm can be started by clicking the button Automatic generating of the keystream. So the keystream is generated and it is possible to decrypt the ciphertext using this keystream.

The keystream contains as many numbers as there are characters in the window of the lastly opened document in CT1. By clicking the button Decrypt the ciphertext can be decrypted at once (but only if the created initial deck is the same one as the initial deck, which was used for the encryption). The found "plaintext" will be shown in a newly created window in the CT1 main window.


With the Cancel button you can leave the dialog at any time.
