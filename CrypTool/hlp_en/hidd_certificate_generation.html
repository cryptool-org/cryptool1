Dialog Create Certificate and PSE

This dialog provides the PSE and certificate for the RSA signature demonstration (pops up when pressing the button Provide certificate).

The area title Public RSA parameter displays the bit length and public parameters of the key that was generated in Generate RSA key resp. imported using the Import certificate and key button described below.

Please enter Personal data for the certificate as described in the online help page for the Key generation dialog.

The field User Key ID displays the name of the PSE being generated or selected. The field Distinguished Name shows the name of the certificate in the CA data base (CA = certification authority).

An already existing PSE can be used by selecting the Import certificate and key button. That way a previously generated key and certificate an be re-used for further signatures. The PSE can be selected as described in RSA decryption.

If you import a key as described above, then the RSA key generation step (RSA key generated dialog) can (and should) be skipped.

The button Create Certificate and PSE finally, generates a new Certificate signed by the demonstration CA, which is delivered with CrypTool, and stores it together with the previously generated key pair (dialog Generate RSA key) in a PIN protected and encrypted PSE file. Additionally the certificate is stored in the CA data base.
