Dialog Analysis Options

You can reach this dialog via the menu entry Options \ Analysis Options.

This dialog contains several options, which influence procedures in the menu Analysis.

  1. The upper part can be used to specify the output of certain intermediate results for the automatic analysis of different encryption algorithms.

    For the Caesar encryption algorithm the relevant parameters are those shown in the box on the left;
    while for the Vigenère encryption algorithm, Byte Addition and the Exclusive-OR the relevant parameters are those shown in the box on the right-hand side.

    Show further analysis results means with Vigenère analysis, that histogram and correlation are shown for each "column Caesar" text fragment.
    Using XOR or byte addition only the binary histogram window is shown additionally.

    All items which have been checked are displayed. The check marks in the boxes can be removed and restored by clicking with the mouse. The more boxes are checked, the more information windows will be output during the analysis.

  2. In the middle part of the dialog you can alter settings for the Vigenère analysis according to Schroedel: You can switch extensive logging on or off, and you can specify which files are used as statistical data for the analysis.

  3. The lower part of this dialog contains options for Brute-force attacks on symmetric algorithms. These attacks are described in more detail in the online help of the dialog Key Range for Brute-force Analysis.

    The option Number of bytes of the ciphertext document used for the entropy test determines how many document bytes, starting from the beginning, are used in the entropy test. The default value of 128 bytes is a good compromise between speed and accuracy. Smaller values are slightly faster, but might not find the correct key. Higher values can be useful if the document starts with a region of high entropy.
