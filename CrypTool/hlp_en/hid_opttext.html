Text Options (Menu Options)

In the Text Options dialog you can specify which alphabet should be used with the classical encryption procedures.

The default setting for the alphabet is the letters "A" to "Z", case insensitive. The individual characters or character groups (upper case letters, lower case letters, digits, special characters and spaces) can be accepted or deleted either in groups or individually. If a different alphabet is chosen, the classical encryption algorithms will behave accordingly and a corresponding extended translation will be performed.

In the Keep characters not present in the alphabet unchanged field it is possible to specify the response to characters which are not contained in the alphabet. If this is selected (the default), any characters which are not contained in the alphabet will be transferred unchanged into the encrypted or decrypted text. Moreover, if the option to keep upper and lower case is selected, then upper and lower case features will likewise be transferred from the original document.

From the cryptographic viewpoint the retention of text formatting and special characters is not a good idea. However, for demonstration purposes it is nice as it means that a reconstructed plaintext is easier to read and verify.


Remark: Persistency and internal key store

The settings you make in the Text Options dialog (like the alphabet) are not stored -- neither in the internal key store nor in the registry.
So if you restart CrypTool the suggested alphabet for the text based ciphers contains initially the capital letters only.
