Message Authentication Code (Menu Individual Procedures)

The dialogue Message Authentication Code demonstrates how to create the message digest of a file using a hash function combined with a secret key. A message authentication code (MAC) aims at achieving authenticity.

Both parties have to agree on a shared secret key and a cryptographic hash function before exchanging messages. The sending party combines message and key (for example by appending the key) and hashes the newly resulting file. The hash value is transmitted together with the original file.

By using a cryptographic hash function data integrity becomes verifiable, because even changing 1 letter would result in another MAC value. The secret key that is part of the MAC-calculation, authenticates the sender.

To verify authenticity the receiving party performs the same operations as the sending party did before. But instead of the original file, the transmitted file is used. If the calculated result and the received MAC correspond, the receiver can assume that the message derives from the given sender and was not altered during transmission.
