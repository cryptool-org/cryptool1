Dialog Key Entry: Permutation / Transposition

You can reach this dialog via the menu entry Encrypt/Decrypt \ Symmetric (classic) \ Permutation/Transposition.

This dialog is used to enter the permutations for Encryption by permutation.

In each of the two input fields a keyword or a numerical permutation can be entered. The method for working out the relevant permutation is described under Encryption by permutation.

An entry in the first field is mandatory. An entry in the second input field is optional. If a second entry is made not one but two permutations are performed.

For each permutation it is possible to specify whether the data should be read in/out by row or column. Additionally it is possible to set the permutation to be a permutation of columns or a permutation of rows.

The maximum permutation length is 100. It can be entered either as 100 capital letters or as a number sequence that is a permutation of 1, 2, ..., 100.

By activating the checkbox within the group Options you can determine that before encryption or decryption the input permutations are inverted. Furthermore you can specify how the input data is processed: While all characters are processed in binary mode, text mode works differently: Depending on the current settings in the text options dialog (see button Text Options), characters which are not part of the current alphabet are either retained without change (and their positions are respected) or removed before the actual transposition.

You can get information on the Used permutation and its inverse by clicking the Encrypt or Decrypt button.

Remark 1:
Historically the permutation exchanging columns twice was called double column transposition.

Remark 2:
The first historically know variant of this cipher was the Scytale of Sparta (500 B.C.).

Remark 3:
For single step permutation a known-plaintext analysis is available.
