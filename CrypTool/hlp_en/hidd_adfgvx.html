Dialog key input ADFGVX-Cipher (Menu Crypt/Decrypt \ Symmetric (classic))

In this dialog window, the key for the ADFGVX encoding method is entered.

Since ADFGVX is a two-stage cascade cipher, the key consists of two parts: the substitution key and the transposition key. Accordingly, the key input dialog window is divided into two sections.

Stage 1: substitution

The substitution matrix is filled with each letter from 'A' to 'Z' and each cipher from '0' to '9', other entries are not allowed. On repeated entry of an existing character, a dialog box is displayed, giving the user the possibility to erase the old entry or to undo the change. The button Standard matrix fills the substitution matrix with all characters in order, the button Random matrix fills it with a random order of all characters. Clicking the button Clear matrix and confirming the pop-up-dialog will erase the substitution matrix. This is a comfortable way to enter examples from literature or other sources.

Stage 2: transposition

The text field Transposition password can contain any string of characters. Redundant or invalid characters will be removed before de- or encryption. Below the text field, the current sequence for the column transposition is displayed.

Output options

By checking the box Separate text by blanks and entering a value in the adjoining input field, it is possible to display the ciphertext in groups of 1 to 26 characters. The default value "5" is determined by the historical circumstances: during the Great War the ADFGVX-code was transmitted via morse code; the morse signals were sent in groups of five letters.
The check box Additional line feed after each block starts a new line of text after each group of letters.

The box Output stage 1 causes the unpermutated but substituted text to be written to a separate document. This output can be formatted in the same ways as the final ciphertext. By default, the block length is adjusted to the transposition password length

To enhance the readablility of the ADFGVX-bigrams in stage 1, a grouping of two letters is advisable. If you want a better understanding of the column transposition, you should set the grouping of both stages to the current password length and activate the new line option of stage 1. You will discover that the text of stage 1 read by columns from top to bottom is identical to the final ciphertext read by the line.

If both parts of the ADFGVX key are entered correctly, the button Encrypt is activated. You can now encrypt the document with the ADFGVX encoding method.
If the document is already an ADFGVX-encrypted text, the button Decrypt is activated as well and you can decipher the message.

Clicking the key icon inserts an ADFGVX key that was previously copied to the key memory (if the button is inactive, there is no ADFGVX key available).
