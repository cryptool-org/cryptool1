Dialog Hill Analysis (Known Plaintext)

You can reach this dialog via the menu entry Analysis \ Symmetric Encryption(classic) \ Known Plaintext \ Hill.

The data required for the automatic analysis (Known Plaintext attack) for the Hill encryption algorithm must be entered in this dialog.

  1. Plaintext and cyphertext text files.

  2. Multiplication type. Were the encoded plaintext blocks multiplied from the left side by the Hill matrix [(vector row) x (matrix)] or from the right side [(matrix) x (vector column)].

  3. Encoding offset of first alphabet charcter ('0' or '1'). Consider if the set alphabet characters were encoded to the numbers 0, 1, 2, 3, ... or - starting with '1' - encoded to the numbers 1, 2, 3, 4, ... .

  4. Hill dimension range (from, to). You have the option of limiting the dimension of the key to be searched for. If you know the length of the key, then you can instruct the program to confine the analysis to this key length. This can save a lot of time, especially if the document is large. However, you should note that, for example, that this will mean that a key of dimension 3 will not be found if you limit the dimension to 1 or 2.

To start the analysis, click on Continue; to terminate the analysis click on Cancel.

If CrypTool has been able to work out the key it will be displayed in the Hill Key Matrix ; otherwise a message stating that the key could not be found will appear.

Remark 1: Uniqueness of the analysis
For the selected plaintext/ciphertext pair there may exist more than one valid Hill key. But only one key will be shown. In case you need to find a different Hill key, you can start the Hill analysis again after you deselected step-by-step the parameters of the previous found Hill key.

Remark 2: Example for deselecting a parameter
We assume CrypTool found as key TO LM, and the first alphabet character has the value 1. You now can reduce in the options the parameter "Value of the first character of the alphabet" to the value 0, and get the key SN MN.
