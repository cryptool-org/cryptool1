Single Column Transposition (Menu Analysis \ Symmetric Encryption (classic) \ Known Plaintext)

A known-plaintext attack is available for the Permutation encryption algorithm.
This means that it is necessary to enter not only the text encrypted by the permutation encryption algorithm but also the according plaintext. CrypTool then works out the key used to encrypt the plaintext.

For the automatic analysis both the plaintext and the encrypted text are required. Therefore you have to upload both texts using Dialog Analysis of a Single Column Transposition (Known Plaintext). Corresponding texts have always the same length.

The analysis operates as follows: The plaintext as well as the ciphertext will be read into a table. Depending on the parameters (variant) the respective table has n columns or n rows (n = length of permutation). CrypTool compares the columns of both tables and determines the transposition of the columns. This permutation is the key that was used for permutation encryption (together with the parameters of the method).

CrypTool runs through an automatic analysis when the user does not specify the key length and parameters for computing the key. Several keys could be found with it. However, only the first key which is found will be displayed in Dialog Single Column Transposition Key. The cryptanalysis program assumes, that all rows are filled completely. So if a user limits the key size, only such keys can be considered, where the texts have a length of an integer multiple of the keys.

Exemplary computation of the permutation key:

The plaintext is: ABCDEFGHIJKL.
The ciphertext is: IEAJFBKGCLHD.

The following exemplary analysis of a single column transposition expects the following parameters:

Read in: column by column
Permute: column by column
Read out: row by row
Key size: 3

As described above the analysis starts with reading the plaintext and ciphertext into a table. Because of the key size (3) and the column by column permutation, both texts consist of exactly 3 columns. The column size will be calculated by the text size and by the key length. In this example the 3 columns consist of 4 characters, because the text have a size of 12.

First the plaintext will be read into a table column by column.  

A E I
B F J
C G K
D H L

After that, a table of the ciphertext will be created. Because the assumed parameters say the plaintext will be permuted column by column and read out row by row, the ciphertext has to be read in row by row, The ciphertext table looks as follows:  

I E A
J F B
K G C
L H D

CrypTool compares the columns of both tables with each other. Based on this comparison you can see the first column of the ciphertext table is located on the third position of the ciphertext table. The column in the second position kept its location, the third was switched to the first position in ciphertext.
Thereby the analysis of permutation encryption gives the permutation key: 3,2,1.  

Remarks:

1) The known-plaintext attack requires both the plaintext and the encrypted text. The encrypted text can be generated in CrypTool via the menu De-/Encrypt \ Symmetric (classic) \ Permutation/Transposition. The automatic analysis of a single column transposition computes the key used to encrypt the plaintext.

2) The maximum length of both texts is limited to 2 Gigabyte. This comes from the range of values of a 32-bit integer. This data type was used to declare the file size (= text length) and it also determines the limit of the file size. Due to the fact that the key length can not exceed the text length, the permutation size have the same upper limit.
