Dialog Automatic Substitution Analysis 2 - Options

You can reach this dialog after selecting method 2 in the dialog selecting a method for automatic substitution analysis.

In the upper part of the dialog you can decide between automatic or manual analysis.

Options for manual analysis

You can choose one of the supported languages: German or English. CrypTool is rolled out with word lists containing the most frequent words of the respective language. You can modify the word lists with a simple text editor. The only constraint is: words longer than 20 characters are ignored.

If no further options are chosen, then a pattern search is performed using the most frequent words of the currently selected language. Satisfactory results are obtained with this tool when applied to English language texts. However, the analysis results are often much less impressive when the language is German.

For this reason additional options can be chosen.

  • "Use 'e' as most frequent character" option

    Prior to the pattern search, the character that occurs most frequently in the text is worked out. It is assumed that this character corresponds to the plaintext character "e". The pattern search is started having made this (partial) substitution.

    The assumption that "e" is the character found the most frequently in a text is equally valid in German and in English. However, this assumption can be wrong if the text is extremely short or highly specialised.

    Note: A complete (and meaningful) novel has been written in the French language in which the letter "e" does not occur once. This novel has been translated into German (once again, avoiding the letter "e").

  • "Extended Analysis (German plaintext)" option

    This analysis is based on peculiarities of the German language. In principle, however, this analysis would also be possible for English plaintext, albeit with different baseline numbers.

    Prior to the pattern search four letters are assigned in the following manner:

    • It is assumed that the letter that occurs most frequently in the text is the encrypted version of the letter "e".
    • For this reason it would be inconsistent to select this option but at the same time disable the option "Use "e" as most frequent character", so the program does not permit this combination of options.
    • A list containing the letters which occur most frequently at the beginning and end of a word and before and after the letter "e" is created. This is sorted in descending order.
    • The program then ascertains the two letters which in the sorted lists of letters which occur the most frequently at the end of a word and after the letter "e" are nearest to the top of the list (the sum of the indices is minimal). It is assumed that these two letters corresponds to the encrypted letters "n" and "r".
    • The encrypted version of "n" is assumed to be the letter which occurs most frequently at the end of a word.
    • Lists containing the most frequent letters before and after the "n" are then created and sorted.
    • The letter which in the three lists of the letters which occur the most frequently before and after the "e" and before the "n" the greatest number of times is assumed to be the letter "i".

A pattern search is now started having made this (partial) substitution.

Options for manual analysis

Note that the check box Manual analysis using the stored key is inactive unless there exists a suitable key in the key store.



When CrypTool has completed its automatic analysis or when a manual analysis was chosen, a Manual Post-processing window containing the results appears.

In the Examples chapter there is an example of an attack on the encryption algorithm.

Conditions which the document must satisfy for a successful attack

Generally one can say that the key (i.e. the substitutions or at least a large part of the substitutions) can be found if the encrypted document contains several hundreds of characters of German and English text which by and large contain the words which occur most frequently in the language. The words which occur most frequently in the language concerned are held in a list within CrypTool. The analysis does not work if the text to be analyzed consists largely of words which do not occur in the list.

For example, encryption of

THE QUICK BROWN FOX JUMPED OVER THE LAZY DOGS BACK

produces (without manual post processing or extended analysis) the result

thL THEIR brown fox SHOULD ovLr thL MANY Dogs bAIR.

On the other hand, however, remarkably good results can be achieved with this kind of analysis even for very short encrypted texts.

An encrypted version of the text

THIS IS A TEST

produces (without manual post-processing or extended analysis) the result

THIS IS A TeST.

Such a text would be much too short to contain statistically significant information, as required by the analysis methods that are based on frequency distributions. There is a high probability that such an analysis would fail.

The results of a pattern search analysis are significantly poorer for German plaintext than for English plaintext. This is probably due to all the inflexions that are found in German words. Whereas in English the article "the" only appears once in the list (for English plaintext), in the list for German plaintext we find not only "der" but also "die", "das", "den", "dem", etc.

For this reason CrypTool also offers the Extended Analysis option described above, which enables the key to be computed, building on the pre-analysis by CrypTool.
