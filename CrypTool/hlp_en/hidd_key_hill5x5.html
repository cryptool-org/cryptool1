Dialog Key Entry: Hill

You can reach this dialog via the menu entry Encrypt/Decrypt \ Symmetric(classic) \ Hill.

This dialog is used to enter the key for the Hill encryption algorithm.

After you have chosen the size of the key in the Dimension box and input the key, you can encrypt the document by clicking on the Encrypt button or pressing the RETURN key. If you wish to decrypt the document, just click the Decrypt button.

A key that is stored into the key memory could be inserted by the button (if there is no key stored, the button is deactivated and appears in gray).

You may edit the matrix by yourself. Choose either the character matrix or the number matrix - the values between the two matrices will be synchronized automatically (note: negative numbers are not supported). The first character of the selected alphabet starts either with the value 0 or with the value 1. Note. A Hill key is only valid if the number matrix is invertible. Make sure that the selected matrix fulfil this requirement.

Clicking the button Generate random key generates an randomized valid (invertible) Hill key.

It is possible that you could enter a key that was not in fact suitable because the matrix corresponding to the key is not invertible. (For further information, see explanation on the Hill encryption algorithm.) In this case, a "Key is unusable" message box appears. Another key must be selected.

You can define the used alphabet with the button Text options.

With the button Further Hill options you can

  • adjust the value (0 or 1) of the first character of the selected alphabet,
  • adjust the Hill encryption padding character (if the length of the plaintext does not devide the Hill key length the plaintext will be padded by this character).
