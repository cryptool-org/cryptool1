Secret Sharing by CRT (Menu Individual Procedures \ Chinese Remainder Theorem Applications)

By choosing the dialog window Secret Sharing by CRT  you will find an interactive visualization of a "Secret-Sharing-Threshold-Scheme".

It is a concept for distributing keys to an arbitrary number of parties.

Characteristic for this distribution is, that a (before specified) smaller number k < n of keys can rebuild the secret.

For example, you're able to generate keys for 5 parties and can rebuild the whole secret by cooperation of any three of these five.

Description of the procedure

  1. Define the number of parties (n)
  2. Define the number of parties required for the solution (k)
  3. Choose t different, preferably large prime numbers mi (i=1 ... n)
  4. Compute min(k), the product of the k smallest prime numbers
  5. Compute max(k-1), the product of the k-1 largest prime numbers
  6. Compute w, the quotient of min(k) and max(k-1), rounded down to an integer
  7. Choose a random secret s with 0 <= s < w
  8. Choose a random a with 0 <= s + a * w < min(k)
  9. Compute s' = s + a * w
  10. Compute si = s' mod mi (i=1 ... n)

Remark

The Secret Sharing algorithm presented here is not applicable to k=n. If k=n, the secret always equals to the password of the first participant because:

w = min(k) / max(k-1) = m1*m2*m3*...*mi / m2*m3*...mi = m1
s' = s + a * w

The share of the first participant is

si = s' mod w = s' mod m1 = s + a * m1 mod m1 = s

because s<w and a∈N.

This procedure requires at least k participants to reconstruct the secret, and since this requirement is not met for k=n, k=n should not be used. Hence there's an additional requirement in k<n.
