Dialog Attack on the hash value of the digital signature

You can reach this dialog via the menu entry Analysis \ Hash \ Attack on the Hash Value of the digital Signature.

The attack based on the birthday paradox is a generic attack on the hash value of the digital signature, i.e. it can be applied on all hash functions. The attacker‚s objective is to find two different messages that translate into the same hash value.

He designs a harmless message which will be digitally signed by his victim, and a dangerous message which would never be signed due to its malicious content. Afterwards, the attacker generates a great number of modifications of the two messages whose contents are equivalent to those of the original messages and only differ from some extra blanks or unprintable characters.

If the attacker finds a pair of a harmless and a dangerous message with the same hash value he asks his victim to sign the harmless message digitally and subsequently exchanges the two messages.

In the categories Choose "harmless" file and Choose "dangerous" file two messages are determined which are to be used for the attack.

The attack can be configured in the dialog Options for the attack on the hash value of the digital signature.

To start the attack push Start search.

Remark 1:

If you choose two identical files, then CrypTool also tries to generate two different files having the same hash value by adding different patterns.

Remark 2 (CrypTool included example files):

The CrypTool package contains the example files "original.txt" and "fake.txt" as an example of a harmless message and a dangerous message. In order to restore the default paths of these examples you can delete the rows 

SignatureAttackHarmlessFile

SignatureAttackDangerousFile

within the CrypTool registry file. Afterwards the paths of these examples will be shown.
