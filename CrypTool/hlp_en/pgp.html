What is PGP, OpenPGP, GnuPG, Gpg4win ... ?

PGP is short for Pretty Good Privacy. As OpenPGP it is the de-facto standard for privacy and secure communication in the Internet in the private area. The software package PGP was originally developed by Phil Zimmermann and provides strong encryption and digital signature for emails and files. The first version of PGP was available as early as June 5th 1991. It is well-known for helping to defend the right and freedom of private electronic communication.

In the following the (changeful) history of PGP and the current open-source products are explained.

Status in June 2002:

a) The company NAI had bought PGP and developed the Windows version. PGP now was a commercial product and no open-source product any more. After releasing the client version 7.03 NAI stopped developing and supporting the program any further. NAI neither could find a new owner nor were they willing to give their version to the Internet community as open source till then. Phil Zimmermann said, now (his) PGP is accessible for no one anymore – captured in a "prison of intellectual property".

b) Because no PGP source code had been released since version 2.6.3, the open-source project GPG = GnuPG (GNU privacy guard) was started. GPG developed the open source program which also implements the OpenPG standard (RFC 2440), which was developed based on PGP, and is therefore compatible to PGP. GnuPG version 1.0 was published in 1999 and, till today, it founds the basis of other open source PGP packages.
URL: www.gnupg.org

c) The German government [German Federal Ministry of Economics and Technology (BMWA/BMWi) and the German Federal Ministry of the Interior (BMI)] started to fund the command line-oriented project GnuPG (www.gnupp.de) – with the goal to offer a package named GnuPP increasing the user friendliness, especially on the Windows platform, by adding a graphical user interface and plug-ins.

Status in February 2003:

a) The new company PGP corporation, of which Phil Zimmermann is also a stake holder, bought PGP back. An English version of PGP 8.0 was released for Windows (incl. Windows XP) and Macintosh (Instructions in German can be found at www.helmbold.de/pgp). There is a version for non-commercial usage and the sources are also published again. Phil Zimmermann emphasized in a talk given at January 30, 2003: As long as he can decide, the source code will continue to be published and there will be no intended trapdoors within PGP.
URL: www.pgp.com

b) The current version of the open source package GnuPP is 1.1 – also available for all Windows versions (including Windows XP). It is based on GnuPG 1.06 (the base system GnuPG already achieved version 1.2.1 and is also available for Linux).
The package GnuPP can encrypt files, e-mails, the clipboard or the content of the active window.
GnuPP can be used as freeware both privately and commercially. The software offers a German and an English interface with two introductions: one for beginners and one for experienced users. The assistant GPA is especially helpful for users without crypto-experience in order to generate and manage keys.
Same as PGP 8.0, GnuPP does deliver many plug-ins for email programs (like Outlook, The Bat, Eudora and Pegasus).

Status January 2004:

The GnuPP initiative is no longer active. The last update of the project web site dates September 2003. The successor of GnuPP is Gpg4win (see below).

Status June 2012:

a) Since 2005 release 9.0 of PGP Desktop is available. This release features instant messaging (AOL) communication encryption and supports two factor authentication via smart cards. PGP Corporation spreaded its product line further (e.g. PGP Whole Disk Encryption or PGP Support Package for BlackBerry) and build up the so-called PGP Encryption Platform.
In an official letter to the PGP Community CEO, Philip Dunkelberger, strengthens PGP's commitment to continue to publish the source code in order to enable the community to conduct code reviews and to maintain customer trust.
However, the source code is mainly available for the product PGP Desktop, and even there the build tools are not included in the package. These build tools can only be obtained at charge via PGP Professional Services. With regard to the available sources PGP speaks of more than 3.000 downloads per month.
Besides the founder Phil Zimmermann, PGP's technical advisory board consists of some known experts such as Bruce Schneier or Dr. Crispin Cowan. In June 2010 Symantec Corporation acquired PGP.
Meanwhile, there are free versions with a similar functionality as the PGP from PGP Corporation.

b) The GnuPG project (www.gnupg.org) develops since 1997 the free basis software (the actual signing and encryption program) as a command line program for several operating systems.
The GnuPG project is very active and continuously publishes new versions: Since March 2012 the GnuPG versions 1.4.12 and 2.0.19 are available.
Here you can find further information about GnuPG.

c) There are two constantly active open-source projects which integrate the complete package of GnuPG into graphical front-ends. Additionally they offer installation packages for Windows (and thus are completely replacing GnuPP and are as well a valid replacement for the according commercial PGP software):

  • Gpg4win:
    Gpg4win is a free and complete substitute for the original PGP for the operating system Windows. The installation package contains programs and documentation for file encryption, key management and secure e-mail.
    It is available in English and German.
    It includes exhaustive manuals, also for beginners.
    Current version: 2.1.0 (March 2012).
    www.gpg4win.org (mature and stable project, supported by the German Information Security Agency BSI)
    • The program Kleopatra can be used to manage X509 certificates as well as PGP keys.
    • The Outlook plugin GPGol allows encryption via PGP and S/MIME.
    • The Windows Explorer plugin GpgEX enables crypto operations on files.
  • GnuPG Pack:
    GnuPG Pack is also a free and complete substitute for the original PGP. The installer contains the encryption program GnuPG, the frontend WinPT Tray for key management, documentation and plugins for different mail clients plus Stunnel with OpenSSL.
    It is available in English and German.
    This package exists since 2003 and delivers very regular updates.
    Current version: 12.5.3 (May 2012).
    http://home.arcor.de/rose-indorf/

d) The project Enigmail (http://enigmail.mozdev.org) offers a widely used plug-in for the mail client Thunderbird. This is useful for secure e-mail establishing both the S/MIME and the OpenPGP standard (offering encryption and signature of e-mails).
For the variant OpenPGP, again GnuPG is used.
Current version: Enigmail v1.4.2 (April 2012) for Thunderbird 12.0 and 13.0.

e) A new and not yet completed project GPG4Browsers (http://gpg4browsers.recurity.com/) started in November 2011 to implement the OpenPGP message format in JavaScript. It is currently available as Chrome extension and can integrate itself into Googlemail. Supported functions are encryption and decryption, signing and verification of signatures, and import and export of certificates.
Current version: GPG4Browsers-v0.1.zip (May 2012).
