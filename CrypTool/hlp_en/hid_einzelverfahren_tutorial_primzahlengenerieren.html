Dialog Prime Number Generation (Menu Individual Procedures \ RSA Cryptosystem)

You can access this dialog via the menu entry Individual Procedures \ RSA Cryptosystem \ Generate prime numbers.

Prime numbers are natural numbers greater than 1, which are divisible only by 1 and by themselves, and in cryptography they play an important role. In this demonstration you can generate prime numbers yourself, the prime numbers are generated randomly within freely definable limits.

If you press the Generate prime numbers button, a random search for the prime numbers p and q will be initiated. A number X will be taken at random from the range specified [Lower limit, Upper limit]. The system then "tests" whether X can be a prime number.

For this test random is also necessary:

  • If X is not a prime number, then the chosen test states with a probability of ½ that X is not a prime number, whereas in the other case the feedback is always positive. If the prime number test is repeated 80 times "independently" and always has a positive outcome, then the probability that X is not actually a prime number is extremely small at 2^(-80) and the number thus found is understood as a prime number, and is displayed as either prime number p or prime number q.
  • If one of k prime number test is negative, then X is not prime and the test is repeated with a new randomly chosen number X.

In this dialog you can choose between the following prime number tests:

  • Miller-Rabin test,
  • Solovay-Strassen test and
  • Fermat test

If no prime number has been found after trying 1000 randomly drawn numbers X, then an appropriate error message is displayed. In that case you need to raise the range defined by the limits [Lower limit, Upper limit] for the prime numbers accordingly.

Detailed information on the subject of prime numbers will be found in the CT Book.

Generation of all prime numbers in a specific value range

If the radio button Generate all prime numbers in a value range is checked, the dialog searches for all prime numbers within the value range specified for the prime number p. In contrast to the generation of only two prime numbers, the numbers are not randomly chosen from the value range. Instead, each number within the value range is tested one after another. Please note that this process can be very time consuming. However, you can interrupt this process using the emerging progress bar.

When the process is finished, all prime numbers that were found are dumped into a new document. You can configure the separator used to separate the prime numbers using the input field Separator for the display of the primes.
