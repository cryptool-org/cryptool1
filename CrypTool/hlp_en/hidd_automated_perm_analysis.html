Dialog Analysis of a Single Column Transposition (Known Plaintext)

You can reach this dialog via the menu entry Analysis \ Symmetric Encryption (classic) \ Known Plaintext \ Single Column Transposition.

This dialog is used to enter the needed data for automatic analysis (known-plaintext attack) for encryption by permutation.

You have to select the plaintext and ciphertext into this dialog.

Using permutation encryption the text is normally entered row by row in a table of width n (= length of permutation). The ciphertext is now read column by column in the sequence defined by the permutation. Compared to the classic variant, in CrypTool it is possible to read in and read out the plaintext in either columns or rows. Additionally it is possible to determine if the permutation itself is to be carried out line by line or column by column. Column by column is the default here. In the literature sometimes a line by line notation is used. If all three options are toggled the result does not change, as this constitutes an equivalent setting.

You have the option of limiting the dimension of the key to be searched for. If you know the length of the key, then you can instruct the program to confine the analysis to this key length. This can save a lot of time, especially if the document is large. However, you should note, this means e.g. a key of dimension 3 will not be found if you limit the dimension to 1 or 2.

To start the analysis click on Continue; to terminate the analysis click on Cancel.

If CrypTool was able to find the key, it will be displayed in the dialog Single Column Transposition Key; otherwise a message that the key could not be found will appear.
