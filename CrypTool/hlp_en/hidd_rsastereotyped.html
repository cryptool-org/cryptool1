Dialog Attack on Stereotyped Messages

You can reach this dialog via the menu entry Individual Procedures \ RSA Cryptosystem \ Lattice Based Attacks on RSA \ Attack on Stereotyped Messages.


The dialog Attack on stereotyped messages offers two different approaches:

1. Generate example input data and start an attack
Here you can generate an own example in the dialog which then can be attacked. In addition, it has to be marked which part of the message is unknown. This mode leaves unexperienced users with the option to try out the capabilities and the limits of the attack.

2. Enter all input data manually and start the attack
By entering the public key, the known part of the message, and the ciphertext the attack can be launched. After offset and length of the unknown part are provided by the user, the attack can be started. This mode is not recommended to inexperienced users.


1. Generate example input data and start the attack

  1. First the public key (N,e) has to be entered. e should not be selected too big since the unknown part of the message may not exceed (n/e) bit of the message. Here, n denotes the bit length of N. If e.g. N is a 1024 bit modulus and e equals 3, at most 1024/3 = 341 bit or 42 characters of the message may be unknown.
    It is also possible to make the program generate a random N by entering the desired bit length of N and clicking "Create modulus".
  2. The fastest way to proceed is when the 2nd radiobutton is checked in "Step 2" of the dialog. Then, only the complete plaintext (no ciphertext) has to be entered. With a normal attack, the attacker does not know the complete plaintext, but for educational purposes it is required to generate a valid example. If you want to use a more realistic scenario, use the second approach.
    Remark: In this educational dialog the whole plaintext must fit into one RSA block of N bits. Converted in bytes you get the maximum length in characters shown below the textbox "Plaintext".
  3. Now the plaintext can be encrypted by clicking the "Encrypt" button. The according ciphertext is shown in the field aside.
  4. Either with the mouse or with the arrow keys (holding down the shift key) the part of the plaintext has to be highlighted which should be unknown to the attacker. The highlighting has to be done in in the grouping "Step 2" in the dialog. Make sure that this unknown part is not too long (refer to point 1).
    After clicking the "Cut" button, the textboxes for the known and unknown part of the plaintext are updated, and the "start" button is activated.
  5. To start the attack just click the "Start" button. Section (grouping) "Step 5" shown further information on the current attack. In case of success, the sought und previously unknown parts of the plaintext are displayed.

2. Enter all input data manually and start the attack

First, the public key (N,e) is needed (ignore the field "desired bit length of N").
Then, in "Step 2" of the dialog, the first radiobutton has to be chosen and the complete ciphertext has to be entered.
Then, in "Step 3" of the dialog, the known part of the plaintext has to be delivered. Furthermore, the offset and the length of the unknown part of the message have to be entered. This can be done by entering them directly into the textboxes or using the spin buttons. If the plaintext is already entered, changes can be directly monitored in the "Preview" textbox: Here you see, that "Length" many stars are inserted into the known plaintext -- starting from the given offset.

Attack parameters

In the "Step 4" group, the parameter h can be changed (h>1). A greater value fo h allows a bigger part of the message to be unknown, but an increase in this number results in a bigger lattice-dimension and thus negatively influences the runtime of the program.

To start the attack just click the "Start" button in section "Step 5" of the dialog.
