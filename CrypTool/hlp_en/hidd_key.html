Dialogue Key entry

This dialogue is used to enter the key for the following encryption algorithms:

Caesar

Vigenère

Byte Addition

Binary Exclusive-OR

IDEA

RC2

RC4

DES (ECB)

DES (CBC)

Triple-DES (ECB)

Triple-DES (CBC)

MARS

RC6

Rijndael

Serpent

Twofish

After you have entered the key, you can encrypt the document by clicking on the Encrypt button or pressing the RETURN key. If you wish to decrypt the document, choose the Decrypt button.

A key that is stored into the key memory could be inserted by the button (if there is no key stored, the button is deactivated and appears in grey).

Under most of the symmetric procedures (IDEA, DES, MARS, RC6, Rijndael, Serpent, Twofish) a fixed length key must be used. Shorter keys are padded out with zeros. Example: As DES key you choose 12 AB:

The current file is encrypted with the DES key 12 AB 00 00 00 00 00 00 because the DES algorithm is expecting a key comprising 64 bits of hex characters.

Typical key lengths and their presentations are listed in the table Key presentation.

A few classical procedures require different types of key input dialogues; Hill is a good example to demonstrate this difference.
