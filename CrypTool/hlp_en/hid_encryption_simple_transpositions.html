Scytale / Rail Fence (Menu Crypt/Decrypt \ Symmetric (classic))

The key for these transposition algorithms is entered in the Scytale / Rail Fence dialog.

Transposition ciphers scramble the order of the letters in the cleartext.

Two simple transpositions are the Rail Fence encryption and the Scytale.

Rail Fence encryption

The Rail Fence encryption writes the letters of the cleartext alternately in two or more rows. One can retrieve the encrypted message by reading out the letters row by row.

Below you can find an example of a Rail Fence encryption with four rows. The cleartext is written in the rows from top to bottom. If the last row is reached, the cleartext is written in the rows from bottom to top. In this example we use the cleartext "SECRETMESSAGE":

Reading off row by row yields the message "SESEERTESGCMA". The receiver can decrypt the message by simply inverting the procedure. In order to do that, the receiver needs to know over how many rows the message was spread.

Variations thereof are for example:

  • An additional offset (that means the first letter of the cleartext is not written in the first, but in the third row for example, and from there the procedure continues as usual). In this case, the offset is part of the key.
  • Pairs of letters are transformed instead of single letters. This is very weak.
  • Also, you can create groups of letters of a fixed size and scramble the letters within these groups according to a certain pattern (permutation). If each of the groups consists of only one pair of letters, the first and the second letter switch places, the third and the fourth and so on. You can try this using the menu Crypt/Decrypt \ Symmetric (classic) \ Permutation/Transposition.

Scytale

The Scytale is a transposition cipher as well. It was the first military encryption algorithm, and it was used by the Spartans as early as the fifth century. The Scytale is a wooden stick wrapped by a strip of leather or parchment. The sender writes his message along the wooden stick and then unwraps the strip. After that the strip seems to contain only a pointless series of letters. Possibly, the strip was used by the messenger as a belt with the letters facing the inside. In order to recover the message, the receiver simply wrapped the strip around a Scytale of the same diameter as the Scytale of the sender. In 404 A.D., Lysander of Sparta ran into a dying messenger, one of only five who survived the strenuous march from Persia. The messenger gave Lysander his belt, and Lysander wrapped it around his Scytale. Thus he learned that Pharnabazos, the governor of the king of Persia, planned an attack against him. Thanks to the Scytale, Lysander could prepare for the attack and successfully defend it. A historical overview on this time span can be found here: http://www.uni-kiel.de/klassalt/lehre/materialien/fell/ws2007_gk/grundkursaltegeschichtews200708sitzung05.pdf (German only)

Therefore, the key is a Scytale with the correct diameter or the correct number of edges, respectively. If you can guess the procedure, the Scytale is a very insecure algorithm since the amount of possible keys is limited.

A variant thereof is to use an additional offset: Using an offset of 1, the wrapping of the cleartext is not started on the first winding in the first row, but on the second winding. Consequently, the cleartext does not start at the beginning of the belt. This offset is part of the key, too.

With n edges and no offset, a Scytale is equivalent to a transposition with the key 1,2,3 ... n if input and permutation are processed column by column and output is processed line by line.
