The CrypTool initialization file "CrypTool.ini"

The default settings for some options used in CrypTool to survive a program session (persistance) are stored into the file CrypTool.ini.

Starting CrypTool the first time introducing information (starting dialog and an example file) will be shown. Each time you have the option to decide if this will be shown in the future starts of CrypTool.

The file CrypTool.ini is located in the standard Windows directory (e.g. C:\Windows or C:\WINNT). Below the information stored in CrypTool.ini is explained.

The CrypTool.ini file is organized in sections. Each section starts with an expression in brackets. CrypTool uses the sections

  • [Settings] for generic settings,
  • [Recent File List] for the list of the 10 last files which where opened in CrypTool and
  • [SigAttTest] for parameters used to control the series of experiments of the birthday attack against the digital signature.

Section [Settings]: General Settings

  1. Start options for CrypTool:
    The Start options are set via the menu Options \ Start options.
    NoTipps=1 or NoTipps=0 define if the "How to Start" dialog is shown at the begin of CrypTool.
    SampleTextFile=1 or SampleTextFile=0 defines, if CrypTool opens automatically an example file or if it starts with an empty main window.

  2. Options for the Diffie-Hellman key exchange protocol:
    If you start the Diffie-Hellman key exchange protocol at first time, a short introducing dialog text is shown at the beginning. The variable DH_IntroDialogue in the CrypTool.ini file controls if this dialog will be shown. If you set this variable to 0, this information dialog will be shown no more. Setting this variable to 1 or deleting this variable will lead to the opposite. These settings are also controlled via the menu Options \ Further Options.
    Before each single step of the Diffie-Hellman protocol information to this step will be shown. Setting DH_InfoDialogues to 0 disables the appearence of these dialogs. Setting this variable to 1 or deleting this variable will enable the information again.

  3. Options to the birthday attack on the digital signature:
    The options which are set in the dialog Options for the birthday attack on the digital signature are stored within the CrypTool.ini file as follows:

    The option SignatureAttackTestMode=1 controls that instead of the interactive usage a series of experiments for the birthday attack is performed (see below: series of experiments for the birthday attack against the digital signature). By default this option has the value 0 for interactive usage.

    The option SignatureAttackHashAlgorithmID stores the hash function (0=MD2, 1=MD4, 2=MD5, 3=SHA, 4=SHA-1, 5=RIPEMD-160) which should be attacked. The option SignatureAttackSignificantBitLength=32 defines the number of significant bits, which are considerd during the collision search (in this example 32 bits).

    If the option SignatureAttackModificationMethod is set to 1 (which means that during the text modification additional blanks or carriage returns (whitespaces) are included), then the parameters SignatureAttackCheck1 resp. SignatureAttackCheck3 control whether blanks before end of line resp. doubling blanks are allowed during the text modification.
    If the option SignatureAttackModificationMethod is set to 0 (then the text modification takes place in the last row of the text), then the option SignatureAttackAttMethod=0 or =1 defines whether the characters of the text modification are visible (for demonstration) or not.

    Beside the options above the paths of the initial files for the signature attack are stored within the ini file. If you remove the following rows from your CrypTool.ini file then the initial paths of the CrypTool example files are stored within the ini file:

    SignatureAttackHarmlessFile=C:\Programs\CrypTool\original.txt
SignatureAttackDangerousFile=C:\Programs\CrypTool\fake.txt


Section [SigAttTest]: Series of experiments for the birthday attack against the digital signature

Within a series of experiments (series of tests) the attack is done automatically without interaction for different parameters.

A series of experiments for the birthday attack against the digital signature is controlled via the configuration file CrypTool.ini. This requires the variable SignatureAttackTestMode in section [Settings] to be set to 1 and the settings in section [SigAttTest] to to be enhanced by the following parameters:

The directories of the harmless and the dangerous file are set via the variables HarmlessDocument and DangerousDocument.

The parameters SignificantBitLengthMIN and SignificantBitLengthMAX define the start length and the end length of the significant bits for the hash values. The parameter SignificantBitLengthJump defines the step width for each increase of the significant bit length during the test row.

Using the parameters HashAlgorithmIDMIN=0 and HashAlgorithmIDMAX=2 you can determine the hash functions during the series of experiments. The example above (a range from 0 to 2) means that during the experiments against the hash funtions MD2, MD4 und MD5 (0=MD2, 1=MD4, 2=MD5, 3=SHA, 4=SHA-1, 5=RIPEMD-160) are performed.

The parameter Attempts defines how often for each fixed parameter (see above) a test has to be repeated independantly (those replays are valuable for statistic analysis).

The series of experiments is documented by 4 report files:

The results of the birthday attacks are stored in a log file. Additionally the original harmless and the original dangerous file plus the configuration of the series of experiments in further report files. To ensure a unique matching of the report files a numeric time prefix <TIME> is added by CrypTool to the file names. You can define the directory of the report files using the parameter LogPath.

The 4 report files have the following names:

<TIME>_Dangerous.txt // Original text of the dangerous file

<TIME>_Harmless.txt // Original text of the harmless file

<TIME>_Header.txt // Configuration of the series of experiments

<TIME>_Log.txt // Results of the series of experiments

Example:

The following configuration determines the series of experiments for the birthday attack against the digital signature over the files original.txt and fake.txt for significant bit lengths of 32, 34, ..., 40 bit. The hash functions MD2, MD4 and MD5 are used. For each configuration (bit length, hash function) the attack is repeated independently 10 times. This test row applies the birthday attack 5 x 3 x 10 = 150 times.

[SigAttTest]
SignificantBitLengthMIN=32
SignificantBitLengthMAX=40
SignificantBitLengthJump=2
HashAlgorithmIDMIN=0
HashAlgorithmIDMAX=2
Attempts=10
HarmlessDocument=C:\Programs\CrypTool\original.txt

DangerousDocument=C:\Programs\CrypTool\fake.txt

LogPath=C:\results\

Remark 1:

Please keep in mind that all parameters/options (besides the ones for the birthday attack series of experiments) should be set only via the option dialogs within CrypTool. Only there checks against false input values are made.

This documentation here about CrypTool.ini is made above all for developers or "researchers" and less for "normal" users.

Only the ones who do series of experiments with CrypTool must directly edit CrypTool.ini. Then they have to take care to make only correct settings.

Remark 2:

The following parameters are stored in CrypTool during the current session, but not permanently:

  • Passwords used for different encryption methods,
  • Length of the entropy window used at the brute-force analysis.
