Dialog Analysis of the Playfair encryption algorithm

You can reach this dialog via the menu entry Analysis \ Symmetric Encryption (classic) \ Manual Analysis \ Playfair.

The analysis tool for the Playfair algorithm supports the manual analysis of a text encrypted with the Playfair algorithm. As a pre-condition, parts of the plaintext must be available in addition to the ciphertext. Based on this information the key matrix can be constructed.

Using intermediate results the matrix can be guessed or CrypTool can be asked for a suggestion. You can watch the results closely, correct them if necessary and finally try a decryption.

The encryption matrix can be edited via the fields Password and Matrix. An entry in the Password field is written to the matrix, as described in Playfair encryption algorithm. A character entered directly into the matrix exchanges the positions of the entered character with the character at the cursor, if the character at the cursor is not a star (‘*‚).

If the option Update expected plaintext using the key matrix is activated, after each re-computation of the key matrix the plaintext of the actualized decryption is displayed on the input line expected plaintext.

The text displayed in the box labeled Analysis result reflects the effect of the selected matrix.

  • The first row shows the beginning of the encrypted text.
  • The second row equals to your stipulation as entered in the field Expected plaintext (di-gram mappings already determined are applied consistently).
  • The third row displays the analysis result based on the stipulation. It is calculated from the encrypted text and the currently selected matrix.

The constraints for the matrix are shown in the field Letter information. This information is computed from the encrypted text and the assumed stipulation. They have no direct impact on the shown matrix, but give you some hints for suggesting the right encryption matrix.

The table Letter information displays the following information for all letters in the alphabet based on the content of the field Expected plaintext:

  • LROU: These column shows which neighbors the letter has definitely. If there is the combination 'S*V*'‚ in the row for letter 'H', then it's sure that 'S' stands on the left side of 'H'. Otherwise no matrix can match the ciphertext and stipulation. Correspondingly, 'V' is definitely over 'H'. The other neighbors are not determined yet.
  • Nb?: The letters before the delimiter '/' stand left or above the letter, the letters behind the delimiter are on the right side or below of the letter.
  • Horizontal: The listed letters are usually in the same row as the letter in the first column. If '!' follows, it's for sure. If '°' follows the letter, then the letter is in the same row or it's the upper neighbor. If '.' follows, then the letter is in the row or below of the given letter.
  • Vertical: It's sure that listed letters here are in the same column as the given letter.
  • row or col?: This is the weakest constraint, that a letter is in the same row or in the same column as the given letter.

Hint: Normally letters which are listed in row or col but not in horizontal‚ are often in the same column as the given letter.

Clicking the button Create matrix suggest a matrix, which respects the constraints and therefore the stipulation based on the ciphertext. The resulting matrix can be modified. Please note that changes to the matrix get lost if you click on the button.

In the case that your stipulation does not begin on the first position or contains gaps you can put in a placeholder ('*') by typing the space bar. CrypTool ignores all di-grams which contains at least one star in the stipulation.

When you type the stipulation, CrypTool can detect inconsistencies during the calculation for the constraints. The most common error is to type the same letter into the stipulation as the letter occurring in the encrypted text on the same position. Using the Playfair algorithm you can't get a letter for encrypting for itself. (An 'A' will never be encrypted to an 'A'.)

The possible stipulation errors are:

  • The Playfair cipher letter and plaintext letter are always different.
    Because of the design of the Playfair algorithm, a character can never be mapped onto itself. If there is somewhere in the stipulation a letter on the same position in the stipulation as in the encrypted text, this error message occurs. E.g. encrypted text 'ABCD' and stipulation 'QWCA' leads to this error message because letter 'C' on position 3 occurs in both.
    Therefore, the assumption is wrong.
  • Error on the plaintext analysis! At least one of the lists is overflown.
    The letter graph for the constrains contains several lists. By using a 5x5 matrix there could be only 4 letters in the same row of the key matrix. (Letters shown with an '!'). If the calculation of the constrain comes to an result that 5 letters have to be in the same row, this list is exhausted. In the list row or col this message occurs on the 9th letter.
  • Error on the plaintext analysis! At least one of the neighbors was assumed as correct.
    In this case one of the neighbors was already calculated and the following calculation has the result that another letter has to be on this place.

If the dialog is closed via the button Output results of analysis, the current key matrix is used to decrypt the ciphertext and the result is displayed in a new document.
