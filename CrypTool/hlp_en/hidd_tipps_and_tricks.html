What is CrypTool?

CrypTool is a freeware program which enables you to apply and analyse cryptographic mechanisms.

CrypTool contains an exhaustive online help, which can be understood without deep knowledge in cryptography, therefore no printed user manual on how to use CrypTool is provided. CrypTool is completely available in English and German. CrypTool has implemented almost all state-of-the-art crypto functions and allows you to learn about and use, modern and classic cryptography within the same environment.

The methods available include both

  • classical methods (e.g. the Caesar and the double-column transposition (permutation) encryption algorithms) and

  • modern cryptosystems (for example, the RSA and AES algorithms, as well as algorithms based on elliptic curves).

A summary of all the encryption algorithms available in CrypTool is contained on the help page for the Crypt menu.

Automatic analysis tools for obtaining the key, starting from a knowledge of the encrypted document and any additional information (the unencrypted document or the language of the document), are provided for the classical encryption algorithms. For additional information on automatic analysis, go to the help screen for the Analysis menu and choose the encryption algorithm to be analysed there.

To support your own analysis of documents, CrypTool can display a histogram of the document, determine the statistics for any N-grams and calculate entropy and autocorrelation.

In the menu Indiv. Procedures single different methods and protocols are offered:

  • calculate hash values (see menu Indiv. Procedures \ Hash).

  • compress and decompress documents. This enables the study of file compression prior to the actual encryption of a given document.

  • generate or analyse random numbers.

  • find out, how strong keys can be generated according the PKCS#5 standard from passwords.

The further menu items are described below.

The RSA cryptosystem (see menu Indiv. Procedures \ RSA demonstration) is covered comprehensively and implemented for different codings. The RSA key is generated from prime numbers created by the user. Key generation, encryption and decryption can be followed step-by-step.

Factorisation of numbers is also an important application for cryptography. Simple RSA cryptosystems can be easily cracked using the factorisation algorithms presented in CrypTool. Thus you get an idea how long keys of secure systems should be at least.

Within CrypTool most of the modern algorithms are implemented according international standards.

The menus and sub menus of CrypTool are dynamically generated, depending on the opened file‚s type. E.g. you will only find the menu item "XOR" below "Analysis \ Ciphertext-Only" if a binary file is currently selected. Accordingly the item "Hash \ Hash demo" can only be found under the menu "Individual Procedures", if a text file is opened.

An overview about all menu items within CrypTool can be found within the menu tree.

The authors of CrypTool did not implement cryptographic functionality, where very good free programs were available in the Internet: so the Enigma cipher and dictionary attacks against password files (see John-the-Ripper, http://www.openwall.com/john/) are not part of CrypTool yet.

Online help, documentation and tutorials

During the development of CrypTool much care has been taken to ensure that context sensitive help is available at any point by pressing the F1 key. You can get a fast overview, if you open a file and move through the menu items: whenever interested click F1 to see details about this menu item.

The extensive online help explains all basic cryptographic terms, contains a short list of references and provides a historic overview of the development of cryptography within the time-table.

The examples (tutorials) provided with the online help will make it easy for you to get up to speed.

A big support for raising a deeper understanding of the following topics are the interactive demonstrations (visualizations):

-creating an electronic signature,

-processing hybrid encryption,

-how changes to a document change the hash value

-sensitivity of hash algorithms,

-generation of hash value collisions (birthday paradox) and

-processing the Diffie-Hellman key exchange.

Beside the online help CrypTool provides 4 additional files with documentation:

-The readme file (readme-en.txt) contains all important elements of the CrypTool package and a description of the project.

-In the script (provided as a PDF file with more than 100 pages) you will find more, somewhat mathematically oriented information

-on cryptographic methods,

-on prime numbers,

-on elementary number theory with many examples and

-on current results of the cryptanalytic research.

-The CrypTool presentation with over 30 slides gives an overview about the CrypTool project.

-Also included is the story The Dialogue of the Sisters by Dr. Carsten Elsner. Within this fantasy story a variant of the RSA cryptosystem is used.

Organizational

CrypTool was developed jointly by companies and universities to provide an adequate teaching aid and for the end user awareness programme aimed at raising the IT security awareness of staff and helping them to gain a deeper understanding of the concept of security.

Another objective was to demonstrate the cryptographic techniques employed in Deutsche Bank. Thus it is possible with CrypTool, as a reliable reference implementation of the various encryption algorithms, to test the encryption used in other programs. With this tool it is therefore possible to check whether claims that a program encrypts appropriately are founded or not.

The hash functions, the symmetric and asymmetric encryption algorithms and the key management facilities are based on the Secude toolkit (version 5.4.15) developed by Secude GmbH.

The enclosed Secude library is restricted through the ticket file to a key length of 768 bits for the asymmetric algorithms and smart card support is disabled. If you have a Secude library licence you will also be able to use longer key lengths with the asymmetric methods with the aid of your ticket file.

In order to demonstrate the RSA cryptosystem and to factorise integers CrypTool uses the long integer arithmetic of the Miracl library (version 4.4.3) from Shamus Software Ltd. (http://indigo.ie/~mscott/). We restricted the integer bit length up to 1024.

In order to demonstrate the hash-collisions (using the birthday paradox) the OpenSLL library libeay32.dll is used.
