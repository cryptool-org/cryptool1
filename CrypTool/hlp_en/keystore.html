The Internal Key Store

In CrypTool a clipboard-like internal key storage is implemented which can save exactly one key for each encryption method.
The keys in there are stored not persistently (within the registry) but only during the lifetime of a CrypTool session.

The internal key storage stores the key and the settings (parameters) which are necessary for the particular cipher method (e.g. for Playfair not only the password is stored but also the separators and who doublicate letters are handled, as these configuration parameters are needed to be able to uniquely reverse the operation).
Normally the alphabet is not stored (see text options).
Exception: For the Hill cipher the internal key store considers the alphabet and the text options used with this method as part of the key, because the modul, an important parameter of the method, is created from the alphabet.

a) How to display and store a key (from within a document)?
If you have encrypted a document, please click the icon "Show key". In the following dialog box please click the button Copy key. Thus the desired key which was used to encrypt the document is stored for the specific encryption method in the internal key store.

b) How to store a key (from within a dialog)?
In some dialogs you can copy the current key into the internal key store by clicking the button Copy key. The functionality is the same as in copying a key from within a document (see above).

c) How to retrieve a key?
If you want to decrypt or analyze an encrypted document, please click the icon "Paste key". Thus the desired key is retrieved from the internal key store and copied into the field for key entry. Note that you can only retrieve keys from the internal key store if you stored a key before for the specific encryption method.


Remark 1: Persistency
If you close CrypTool the contents of the key store is lost.

Remark 2: Text options
The settings you make in the Text Options dialog are not stored in the internal key store.
