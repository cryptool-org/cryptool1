Automatic analysis, one-level permutation encryption method (Menu Analysis \ Symmetric Encryption (classic) \ Known Plaintext)

A Known Plaintext attack is available for the Permutation encryption algorithm .
This means that it is necessary to enter not only the text encrypted by the permutation encryption algorithm but also the same text in plaintext form. CrypTool then works out the key with which the plaintext was encrypted.

For the automatic analysis both the plaintext and the encrypted text are required. Therefore you have to upload both texts into Dialog Permutation Analysis (Known Plaintext). Corresponding texts have always the same length.

The analysis operates as follows: Depending on the chosen parameters, how the texts should be read in, both the plaintext and the ciphertext will be read either row by row or column by column in a table of width n (=length of permutation). CrypTool compares the columns of both tables and determines the transposition of the columns. Subsequently CrypTool determines the key that was used for permutation encryption.

CrypTool runs through an automatic analysis when the user does not specify the data for computing the key. Several keys could be found with it. However, only the first key which is found will be displayed together with the used parameters in Dialog Key Permutation. If the the key size will be limited by the user, the length of the texts has to be a multiple.

Exemplary computation of the permutation key:

The plaintext is: ABCDEFGHIJKL.
The ciphertext is: IEAJFBKGCLHD.

For an exemplary analysis of permutation encryption the following parameters will be expected:

Read in: column by column
Permute: column by column
Read out: row by row
Key size: 3

As described above the analysis starts with reading the plaintext and ciphertext in a table. Because of the key size (3) and the column by column permutation, both texts consist of exactly 3 columns. The column size will be calculated by the text size and key length. In this example the 3 columns consist of 4 characters, that implies a text size of 12.

First the plaintext will be read in a table column by column.  

A E I
B F J
C G K
D H L

After that, a table of the ciphertext will be created. Because the assumed parameters say the plaintext will be permuted column by column and read out row by row, the ciphertext has to be read in row by row consequently. The ciphertext table looks as follows:  

I E A
J F B
K G C
L H D

CrypTool compares the columns of both tables with each other. Based on this comparison you can see the first column of the ciphertext table is located on the third position of the ciphertext table. The column in the second position kept its location, the third was switched to the first position in ciphertext.
Thereby the analysis of permutation encryption gives the permutation key: 3,2,1.
