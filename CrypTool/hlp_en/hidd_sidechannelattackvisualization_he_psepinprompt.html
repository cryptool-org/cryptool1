Dialog PIN prompt for hybrid decryption

This dialog window is shown if you press the Decrypt message button in the side-channel attack against the hybrid encryption protocol dialog.

In order to decrypt the hybrid encrypted message, you need the session key which was encrypted with Bob's public key. For the decryption of the session key you need Bob's private key.

Only Bob is supposed to be able to decrypt the message, therefore Bob's private key is protected by his personal PIN.

Remark:

If you did not execute the hybrid encryption on you own, Bob's personal PIN is 1234 by default.
