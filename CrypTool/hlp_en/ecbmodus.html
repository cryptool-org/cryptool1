ECB mode

ECB stands for Electronic Codebook.

One plaintext block is encrypted after the other. As identical plaintext blocks (with the same key) are also always mapped to identical blocks of ciphertext, one could (theoretically) create a code book with plaintext blocks and the associated blocks of encrypted text (hence the name ECB). However, with a block size of 64 bits the code book would have 2 to the power of 64 = 2 * 2 *... * 2 (64 two‚s) = 18,446,744,073,709,551,616 entries and hence far too many to calculate and store in advance. Moreover, for every key there would be a different code book.

In the CBC mode, every encrypted block is dependent on the previous blocks.
