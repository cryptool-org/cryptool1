Dialog Manual Analysis of Monoalphabetic Substitution (Menu Analysis \ Symmetric Encryption (classic) \ Manual Analysis)

Here you can perform a computer supported manual analysis.

The same dialog pops up, if a ciphertext-only attack for the monoalphabetic substitution encryption algorithm is done, and then you want to manually improve the automated result.

This post-processing window appears once CrypTool has finished its automatic analysis of monoalphabetic substitution or if you directly enter the menu Analysis \ Symmetric Encryption (classic) \ Manual Analysis \ Substitution. The upper section of the dialog shows the letter assignments which have been worked out by CrypTool.

These can be changed by overwriting. For example, assigning the letter A to the letter E means that the letter A in the encrypted text corresponds to the plaintext letter E. CrypTool checks whether any new assignments clash with existing assignments, i.e. whether two different letters have been assigned to one and the same letter (which is not permitted in monoalphabetic substitution). If a collision is identified, an error box asks whether you wish to keep the old assignment or the new one. The assignment which is discarded is set to unknown (*). If the input is valid, it is applied to the text to be analyzed and the results are presented in the lower part of the dialog.

Your own inputs can be reversed by clicking on Reset entries to results of the automatic analysis, upon which the upper part of the dialog once more displays the (partial) substitution worked out by CrypTool. You may undo the last change with a click on the button Back.

Clicking on the Output current status button causes the encrypted text to be decrypted with the current substitution and output in a new text window.
