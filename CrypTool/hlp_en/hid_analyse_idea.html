Automatic analysis, IDEA encryption algorithm (Menu Analysis \ Symmetric)

A Brute-force attack is available for encryption with the IDEA encryption algorithm.

The key space to be searched through is entered in the Brute-force dialogue. After the document has been decrypted with all the possible keys, the plaintext version of the document is assumed to be that which has the smallest entropy in the first thousand characters.

As the IDEA encryption algorithm can only encrypt documents of a particular length, it is always lengthened at the end. As a result it is possible simply by looking at the end to determine whether the key applied during encryption is correct. It can happen that in the entire search range no possible key is found.

In the Examples chapter there is an example of an attack on the Triple-DES encryption algorithm in CBC mode. This example can be worked through for each of the symmetric encryption algorithms; however, the encrypted documents are always different.

Conditions which the document must satisfy for a successful attack

The key will always be found if the document is not too short (at least 100 characters long) and the first thousand characters at the beginning of the document are not distributed too evenly (see Histogram and Entropy). This applies to most documents which are not compressed.
