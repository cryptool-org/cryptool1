Szenario Challenge-Response (symmetrisch)

Dieses Szenario verfeinert die Idee der Einmalpasswörter noch etwas. Die Verbesserung besteht darin, dass das aktuelle Passwort erst während der Durchführung des Protokolls erzeugt wird. Das wird möglich durch die Verwendung eines (symmetrischen) Verschlüsselungsverfahrens.

Das Protokoll beginnt damit, dass Server 1 eine Zufallszahl an den Client schickt, die Challenge (Herausforderung) genannt wird. Der Client verschlüsselt diese Zahl mit dem geheimen Schlüssel und schickt das Ergebnis als Antwort (Response) an Server 1 zurück. Dieser kennt denselben geheimen Schlüssel ebenfalls, entschlüsselt die Antwort und vergleicht sie dann mit der Challenge. Stimmen diese überein, akzeptiert Server 1 die Identität des Clients und der Client ist authentifiziert.

Exkurs Schlüsselaustausch: Wollen n Parteien (Beispiel n=1000) vertraulich miteinander kommunizieren und soll symmetrische Kryptographie so verwendet werden, dass je zwei Kommunikationspartner einen gemeinsamen Schlüssel verwenden, so sind

½ * n * (n-1)

verschiedene Schlüssel notwendig. (Bei n=1000 sind dies 499500 verschiedene Schlüssel)


Bei asymmetrischer Kryptographie würden

2 * n

Schlüssel benötigt. (Bei n=1000 sind dies 2000 verschiedene Schlüssel)

Da alle nur symmetrische Verfahren einsetzen, verwenden Server 1, Server 2 und Client aus Effizienzgründen in diesem Szenario denselben symmetrischen Schlüssel, um miteinander vertraulich kommunizieren zu können. Eine der möglichen Lösungen dieses Szenarios zielt auch genau auf die damit verbundene Problematik ab.

Ihre Aufgabe als Angreifer ist es wieder, von Server 1 als Client anerkannt zu werden. Diesmal gibt es sehr verschiedene Lösungen. Falls Sie keine Lösung finden, können Sie diese hier einsehen.

Symmetrische Challenge-Response-Verfahren kommen z.B. beim digitalen Mobilfunk (z.B. in GSM-Netzen) zum Einsatz.
