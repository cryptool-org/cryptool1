Dialog Zertifikat und PSE erzeugen

Dieser Dialog dient dazu, die für die Signaturerstellung notwendige PSE-Datei und das für die Signaturprüfung nötige Zertifikate im Kontext der RSA-Signaturdemo zur Verfügung zu stellen (wenn Sie den Button Zertifikat bereitstellen drücken).

Im Bereich Öffentliche RSA-Parameter erfolgt die Anzeige der Bitlänge und der öffentlichen Parameter des im Dialog RSA-Schlüssel generieren erzeugten bzw. des mit der Funktion Zertifikat / Schlüssel importieren importierten RSA-Schlüssels.

Die Eingabe der Benutzerdaten für die Erzeugung eines neuen Zertifikates erfolgt wie im Dialog Schlüssel erzeugen.

Im Feld User Key ID wird der Name der zu erzeugenden bzw. gewählten PSE angezeigt.

Unter dem im Feld Distinguished Name angegebenen Namen wird das Zertifikat in die CA-Datenbank geschrieben (CA = Certification Authority = Teil eines Trustcenters).

Mit dem Button Zertifikat / Schlüssel importieren kann eine vorhandene PSE geöffnet werden, um den darin gespeicherten RSA-Schlüssel und das Zertifikat für die Signatur zu verwenden. Das Öffnen einer PSE-Datei geschieht in gleicher Weise wie im Dialog RSA-Entschlüsselung.

Wenn Sie auf diese Weise einen Schlüssel importiert haben, ist das Erzeugen eines neuen Schlüssels im Dialog RSA-Schlüssel generieren überflüssig.

Der Button Zertifikat und PSE erzeugen erzeugt ein neues Zertifikat und legt es zusammen mit dem im Dialog RSA-Schlüssel generieren zuvor erzeugten Schlüsselpaar in einer PIN-geschützten und verschlüsselten PSE-Datei ab. Zusätzlich wird das Zertifikat in die CA-Datenbank eingetragen.
