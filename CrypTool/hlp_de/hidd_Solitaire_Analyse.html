Dialog zur Solitaire-Analyse (Menü Analyse/ Symmetrische Verschlüsselung (klassisch))

Dieser Dialog dient zur Analyse (Solitaire-Analyse) eines mit dem Solitaire-Algorithmus verschlüsselten Ciphertextes. Der Ciphertext muss zuvor als Textdokument geladen werden. Auch hier wird der Text auf 65.535 Zeichen beschränkt und der Rest des Textes wird abgeschnitten. Dann wird anhand des gefundenen Ausgangsdecks ein Initialdeck erzeugt. Mit Hilfe dieses Initialdecks wird anschließend der Schlüsselstrom erzeugt, mit dem der Ciphertext entschlüsselt werden kann.
Bitte folgen Sie den Nummerierungen der umrandeten Felder in diesem Dialog (Gruppierung 1 bis 4).

Ermittlung des Initialdecks

Gruppierung 1: Parameter des gefundenen Decks
Bitte wählen Sie die Art und Weise aus, mit der Sie das "gefundene" Ausgangsdeck bestimmen möchten. Dafür klicken Sie bitte einen der beiden Buttons Ausgangsdeck laden oder Nach Vorgabe an.

  • Ausgangsdeck laden

    • Sie laden ein zuvor bei der Verschlüsselung gespeichertes Ausgangsdeck.
  • Nach Vorgabe

    • Sie erzeugen in der Gruppierung "Nach Vorgabe" ein eigenes Ausgangsdeck. Außerdem kann die Kartenanzahl des gewünschten Kartendecks variiert werden.

Gruppierung: Nach Vorgabe
Dieses Steuerelement wird aktiviert, sobald Sie in Gruppierung 1 "Nach Vorgabe" gewählt haben. Zuerst wählen Sie bitte die Anzahl der gewünschten Karten (3-54). Anschließend definieren Sie das von Ihnen gewünschte Ausgangsdeck, indem Sie die einzelnen Buttons (1-54) in der entsprechenden Reihenfolge anklicken. Unterläuft Ihnen ein Fehler, klicken Sie bitte den Reset-Button und beginnen die Eingabe erneut. Während Sie die einzelnen Buttons anklicken, können Sie ihr bisher erzeugtes Deck im Feld "Gefundenes Deck" sehen. Sobald Sie die von Ihnen gewünschte Kartenanzahl erreicht haben, wird dieses Steuerelement deaktiviert und Gruppierung 2 aktiviert.

Gruppierung 2: Joker positionieren
Aufgrund der Irreversibilität von Solitaire muss an einigen Stellen bestimmt werden, ob Joker A bzw. Joker B hinten oder vorne einsortiert werden sollen. Dieses Steuerelement gibt Ihnen die Möglichkeit zwischen Immer vorne, Immer hinten und Manuell zu wählen.

  • Immer vorne

    • Es wird versucht das Initialdeck zu ermitteln. Dabei werden die Joker bei einem Sonderfall immer vorne eingeordnet.
  • Immer hinten

    • Es wird versucht das Initialdeck zu ermitteln. Dabei werden die Joker bei einem Sonderfall immer hinten eingeordnet.
  • Manuell (=Starten)

    • Es wird versucht das Initialdeck zu ermitteln. Bei einem Sonderfall, wo es zwei Möglichkeiten gibt den Joker einzuordnen, wird der Benutzer um eine Entscheidung gebeten. Mit dem Button Abbrechen innerhalb der Nachricht kann zum Ausgangszustand zurückgekehrt weredn.

    Sowohl bei der manuellen als auch bei der automatischen Auswahl können falsche Wege gewählt werden, die zu einem falschen Klartext führen.

Gruppierung 3: Initialdeck erzeugen
Diese Gruppierung dient zur Darstellung der einzelnen Zustände des Kartendecks.

  • Gefundenes Deck

    • Stellt das gefundene Kartendeck dar, welches entweder selbst eingegeben oder geladen wurde.
  • Zwischendeck

    • Stellt die verschiedenen Änderungen des Kartendecks während der Kryptoanalyse dar.
  • Initialdeck

    • Bei einer erfolgreichen Kryptoanalyse eines mit Solitaire-verschlüsselten Textes wird hier das gleiche Kartendeck angezeigt, wie jenes, welches zur Schlüsselstromerzeugung bei der Verschlüsselung benutzt wurde.

Gruppierung 4: Schlüsselstromerzeugung
Aus dem ermittelten Initialdeck kann nach dem Solitaire-Algorithmus mit dem Button Schlüsselstrom automatisch erzeugen der Schlüsselstrom erzeugt werden. Das Feld unter dem Button zeigt einen möglichen Schlüsselstrom an:

Entschlüsseln
Wenn der Schlüsselstrom komplett erzeugt wurde, kann mit dem Button Entschlüsseln der Ciphertext entschlüsselt werden (vorausgesetzt das erzeugte Initialdeck ist identisch mit dem jenigen, welches zur Schlüsselstromerzeugung bei der Verschlüsselung diente).

Abbrechen

Mit Hilfe dieses Buttons können Sie den Dialog jederzeit verlassen.
