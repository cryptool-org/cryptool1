Dialog Hill-Analyse (Known Plaintext)

Sie erreichen diesen Dialog über den Menüeintrag Analyse \ Symmetrische Verschlüsselung(klassisch) \ Known Plaintext \ Hill.

Die automatische Analyse (Known-Plaintext-Angriff) für das Hill-Verschlüsselungsverfahren benötigt folgende Daten.

  1. Klartext und Geheimtext als Textdateien.

  2. Multiplikationsart. Wurden die kodierten Klartext-Blöcke von links mit der Hill-Matrix [(Vektorreihe) x (Matrix)] oder von rechts mit der Hill-Matrix [(Matrix) x (Vektorspalte)] multipliziert.

  3. Wert des ersten Alpahabetzeichens ('0' oder '1'). Wurden das konfigurierte Alphabet der Reihe nach in die Zahlen 0, 1, 2, 3, ... oder -- beginnend mit der '1' -- in die Zahlen 1, 2, 3, 4, ... kodiert.

  4. Mögliche Dimensionen der Hill-Matrix. Sie haben die Möglichkeit, die Dimension des zu suchenden Schlüssels einzuschränken. Sofern Sie die Länge des Schlüssels kennen, können Sie das Programm anweisen, die Analyse nur für diese Schlüssellänge zu versuchen. Bei großen Dokumenten führt dies zu einer Zeitersparnis. Beachten Sie dabei aber, dass zum Beispiel ein Schlüssel der Dimension 3 nicht gefunden wird, wenn Sie die Dimension von 1 bis 2 einschränken.

Durch Betätigen des Weiter-Knopfes wird die Analyse gestartet; die Analyse wird nicht gestartet, wenn Sie den Dialog mit dem Abbrechen-Knopf beenden.

Sofern ein Schlüssel ermittelt werden konnte, wird er im Dialog Hill-Schlüsselmatrix ausgegeben, ansonsten erfolgt eine Meldung, dass der Schlüssel nicht gefunden werden konnte.

Bemerkung 1: Eindeutigkeit der Analyse
Für das Klartext/Geheimtext-Paar kann es mehrere gültige Hill-Schlüssel geben kann. Es wird immer nur ein gefundener Schlüssel angezeigt. Wenn Sie weitere gültige Schlüssel suchen, dann deselektieren Sie Schritt-für-Schritt die Parameter der gefundenen Schlüssel und wiederholen die Analyse.

Bemerkung 2: Beispiel für das Deselektieren der Parameter
Als Schlüssel wird TO LM gefunden, wobei das erste Alphabetzeichen den Wert 1 hat. Sie können nun in den Optionen den Parameter "Wert des ersten Alphabetzeichens" auf den Wert 0 beschränken und erhalten den Schlüssel SN MN.
