Dialog Primzahlen generieren

Sie können diesen Dialog über das Menü Einzelverfahren \ RSA-Kryptosystem \ Primzahlen generieren aufrufen.

Primzahlen sind natürliche Zahlen größer 1, die nur durch die 1 und durch sich selbst teilbar sind. Sie spielen in der Kryptographie eine wichtige Rolle. In dieser Demonstration können Sie sich selbst Primzahlen erzeugen, wobei die Primzahlen zufällig innerhalb frei definierbarer Grenzen bestimmt werden:

Wenn Sie den Knopf Primzahlen generieren drücken, wird mit Hilfe von Zufall nach Primzahlen für p und q gesucht. Eine Zahl X wird zufällig aus dem Wertebereich [Untergrenze, Obergrenze] gezogen. Danach wird "getestet", ob X eine Primzahl sein kann. Für diesen Test wird wiederum Zufall eingesetzt:

  • Falls X keine Primzahl ist, dann gibt der gewählte Test mit der Wahrscheinlichkeit 1/2 an, dass X keine Primzahl ist, im anderen Fall ist die Rückmeldung stets positiv. Wenn der Primzahltest 80 mal "unabhängig" wiederholt wird und stets positiv ausfällt, dann ist die Wahrscheinlichkeit, dass X doch keine Primzahl ist mit 2^(-80) vernachlässigbar klein und die so gefundene Zahl wird als Primzahl aufgefasst und als Primzahl p bzw. als Primzahl q ausgegeben.
  • Ist einer der k Primzahltests negativ, dann ist X nicht prim und der Test wird mit einer neuen zufällig gewählten Zahl X wiederholt.

In diesem Dialog können Sie zwischen den folgenden Primzahltests wählen:

  • Miller-Rabin-Test,
  • Solovay-Strassen-Test und
  • Fermat-Test.

Für den Fall, dass für 1000 zufällig gezogene Zahlen X keine Primzahl gefunden wurde, erhalten Sie eine entsprechende Fehlermeldung. Sie sollten dann den zulässigen Wertebereich [Untergrenze, Obergrenze] für die Primzahlen entsprechend vergrößern.

Ausführliche Informationen zum Thema Primzahlen finden Sie im CT-Buch.

Generation aller Primzahlen in einem bestimmten Wertebereich

Wenn Sie das Optionsfeld Alle Primzahlen in einem Wertebereich generieren markiert haben, wird versucht, alle Primzahlen zu finden, die sich in dem für die Primzahl p eingestellten Wertebereich befinden. Anders als bei der Generation von nur zwei Primzahlen wird hier nicht zufällig eine Zahl aus dem Wertebereich gegriffen, sondern es wird nacheinander jede Zahl im Wertebereich darauf getestet, ob sie prim ist. Beachten Sie bitte, dass dieser Prozess sehr lange dauern kann. Allerdings können Sie den Prozess mit Hilfe des erscheinenden Fortschrittsbalkens vorzeitig abbrechen.

Ist der Prozess abgeschlossen, werden die gefundenen Primzahlen in ein neues Dokument geschrieben. Welches Trennzeichen für die Ausgabe der Primzahlen verwendet wird, können Sie über das Eingabefeld Trennzeichen für die Ausgabe der Primzahlen einstellen.
