Challenge

We have already calculated optimal results for numbers up to 158 and stored them in the program (including one way to achieve the respective score). For numbers up to 170 we have calculated results with a heuristic method - but we don't know if these results are optimal.

You can help us to upgrade the data we have gathered so far. We'd be pleased if you...

  • find an optimal result for numbers up to 158. You have achieved the same score that we have, but maybe you found a new way to get there.
  • win a game for numbers larger than 158. We have already calculated results for numbers up to 170, but we can't tell if these results are optimal.
  • have an idea for a new algorithm to calculate the highest score possible. Until now we have implemented two algorithms: The first one subsequently trys out every possible solution (brute-force). The second one deals with partial solutions to determine the result (back-tracking). We'd highly appreciate any new approaches in this regard.

If you have mastered any of these challenges, feel free to send your results and comments to esslinger@fb5.uni-siegen.de.

(Status in February 2010)
