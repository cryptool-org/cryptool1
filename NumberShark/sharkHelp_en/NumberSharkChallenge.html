Challenge

Optimal results for numbers up to 158 have been calculated already (thanks to Dan Hoey), and stored in the program (including one way to achieve the respective optimal score). For numbers up to 404 we have calculated results with heuristic methods (thanks to Mr. Schu and Mr. van Nek) -- but we don't know if these results are optimal.

You can help us to upgrade the data we have gathered so far. We'd be pleased if you...

  • find an optimal result for numbers up to 158. You have achieved the same score that we have, but maybe you found another way to get there.
  • win a game for numbers larger than 158. We have already calculated results for numbers up to 404, but we don't know whether these results are optimal.
  • have an idea for a new algorithm to calculate the highest score possible. Until now we have implemented two algorithms: The first one subsequently tries out every possible solution (brute-force). The second one deals with partial solutions to determine the result (back-tracking). We'd highly appreciate any new approaches in this regard.

If you have mastered any of these challenges, feel free to send your results and comments to esslinger@fb5.uni-siegen.de.
